# AIVMT 教师评分操作手册

> **这份手册给"打分"用，不是"采集"用。**
> - 你手上那份《采集日操作手册》是给**学生戴耳麦对着 AI 病人做问诊**（采集对话）用的。
> - **这一份**是给**临床老师（评分员）给已有的问诊记录打分**用的——全程在网页里点选，不需要任何命令行。

**一句话原理**：老师在网页上逐条阅读"学生问诊文字记录"，按 8 个维度打 0–1 分。页面**不会显示任何系统评分或参考答案**（保证盲评）。每条约 8 分钟，共 **42 条**。

---

## 总览：需要谁、花多久

- **评分员：3 位**（你 + 2 位 OB/GYN 同事）。三人**各自独立**打完同样的 42 条。
- **每人用一个固定编号**（如 `fac01` / `fac02` / `fac03`），由你统一分配，**不要重复**。
- **每人耗时：约 5–6 小时**（42 条 × ~8 分钟）。可分多次完成，**中途关掉页面不丢进度**，下次输同一个编号会自动跳过已打的。

---

## 第一部分：开机（只有"操作的那台电脑"做一次，约 1 分钟）

> 这步由你（或会用电脑的人）做。评分老师**不用**碰这步。

### 方式 A —— 老师就在这台电脑上打分（最简单）
1. 在项目文件夹里找到 **`启动评分.command`**，**双击它**。
   - 若第一次提示"无法打开"：在该文件上**右键 → 打开 → 再点打开**（仅第一次）。
2. 会弹出一个黑色文字窗口（正常），并**自动打开浏览器**到评分页面。
3. 让老师直接用这台电脑、这个浏览器页面打分即可（见第二部分）。

### 方式 B —— 老师用自己的笔记本打分（同一 Wi-Fi 下，推荐多人并行）
1. 打开"终端"，粘贴这一行回车（让工具开放给同网段）：
   ```
   cd <AIVMT-repo-dir> && uv run --extra portal python -m aivmt.faculty_portal --host 0.0.0.0 --port 8770
   ```
2. 把这个网址发给 3 位老师（确保他们和这台电脑连**同一个 Wi-Fi**）：
   ```
   http://<本机内网IP>:8770
   ```
   （若换了网络，IP 可能变；用 `ipconfig getifaddr en0` 重新查。）
3. 这台电脑**保持开机、不要合盖休眠**，直到三人都打完。

> 不论哪种方式，老师看到的页面完全一样。下面第二部分是给**老师**看的。

---

## 第二部分：老师怎么打分（评分员照做）

### 第 1 步：输入你的编号
- 打开页面后，在"评分者编号"框里输入**项目组分配给你的编号**（如 `fac01`），点 **进入评分**。
- 同一编号下你已经打过的记录不会再出现——**所以请一直用同一个编号**。

### 第 2 步：读一条问诊记录
- 页面上半部分是一段**学生与"AI 标准化病人"的问诊对话文字**。
- 通读一遍，像平时带教评估学生问诊一样在心里有个判断。
- **页面不会显示任何参考答案或机器分数**——你的判断就是标准。

### 第 3 步：按 8 个维度打分（每项 0–1，可填小数如 0.5）
| # | 维度 | 0 分（差）→ 1 分（好） |
|---|---|---|
| 1 | **开场** Set the stage | 未自我介绍/未说明目的 → 自我介绍、说明来意并建立融洽关系 |
| 2 | **采集信息** Elicit information | 几乎不提问/只问封闭式 → 系统、开放式地引出病史 |
| 3 | **提供信息** Give information | 未给任何解释/反馈 → 用病人能懂的语言清晰说明 |
| 4 | **理解病人视角** Understand perspective | 忽视病人的担忧与情绪 → 主动了解想法、顾虑与期望 |
| 5 | **结束问诊** End the encounter | 突兀结束 → 小结、答疑并妥善收尾 |
| 6 | **病史完整度** History completion | 关键病史几乎全部遗漏 → 应问要点基本问全 |
| 7 | **临床推理** Clinical reasoning | 提问无目的、无鉴别思路 → 围绕鉴别诊断有条理地追问 |
| 8 | **总体表现** Overall | 完全未达标 → 总体表现优秀 |

- 每项填 **0 到 1 之间的小数**（如 0、0.3、0.5、0.7、1）。
- 下方有**备注框（选填）**——可写打分依据或观察，对后续分析很有帮助，但不强制。

### 第 4 步：提交，进入下一条
- 点 **提交并进入下一条**。页面顶部进度（如 `12 / 42`）会增加。
- 重复第 2–4 步，直到出现 **"本评分者已完成全部转写 ✅"**。

### 中途休息 / 换人
- **随时可关页面**，进度已保存。下次重新打开、输**同一个编号**，自动从没打的那条继续。
- 换另一位老师：点页面上方 **切换评分者**，输他自己的编号即可（**绝不要共用一个编号**）。

---

## 第三部分：三人都打完后（你来做）

1. 确认三位老师都看到了 **"已完成全部转写 ✅"**。
2. 评分结果已自动汇总到一个文件：`AIVMT/data/faculty_ratings.csv`。
3. **告诉我"评分完成了"**——我会接管：用系统对同一批记录打分 → 计算与老师的一致性（ICC）→ 把真实结果填进论文。这一步当天就能出第一个真实效度数字。

---

## 出错怎么办

| 现象 | 处理 |
|---|---|
| 双击 `启动评分.command` 提示"无法打开" | 右键 → 打开 → 再点打开（仅第一次需要） |
| 老师笔记本打不开网址 | 确认与主机连**同一 Wi-Fi**；确认主机那行命令还在运行、窗口没关；IP 变了就重查 `ipconfig getifaddr en0` |
| 页面显示"暂无可评转写" | 评分集没加载——联系项目组（我）确认 `data/eval_transcripts/` 有 42 条 |
| 不小心用错了编号 | 没关系，换回正确编号继续即可；用错编号那几条会归到那个编号名下，告诉我清理 |
| 主机要休眠 | 打分期间设为"永不休眠"或定时唤醒，别合盖 |
